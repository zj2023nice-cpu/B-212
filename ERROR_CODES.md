# 奶茶点餐系统 — 全局错误码规范

## 错误响应格式

```json
{
  "errorCode": "A0001",
  "errorMessage": "未授权，请登录",
  "timestamp": "2025-01-01T12:00:00",
  "path": "/api/orders",
  "stackTrace": null
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| errorCode | String | 结构化错误码，格式为 `{模块前缀}{4位序号}` |
| errorMessage | String | 错误描述信息，可包含动态参数 |
| timestamp | String | 错误发生时间（ISO 8601） |
| path | String | 请求路径 |
| stackTrace | String | 堆栈信息（仅 `dev` 环境返回，生产环境为 `null`） |

## 错误码模块划分

| 前缀 | 模块 | 区间 | 说明 |
|------|------|------|------|
| A | 认证授权模块 | A0001–A0099 | 登录、Token、权限相关错误 |
| B | 业务逻辑模块 | B0001–B0099 | 订单、购物车、优惠券、评价、地址等业务错误 |
| C | 商品模块 | C0001–C0099 | 商品、分类、库存相关错误 |
| D | 系统数据模块 | D0001–D0099 | 参数校验、文件、数据库、系统级错误 |
| E | 外部服务模块 | E0001–E0099 | 第三方服务、短信、邮件、支付相关错误 |

## 错误码使用规范

### 后端使用

```java
// 抛出业务异常（使用默认消息）
throw new BusinessException(ErrorCode.C0003);

// 抛出业务异常（使用自定义消息）
throw new BusinessException(ErrorCode.C0003, "商品 [珍珠奶茶] 库存不足，当前库存: 5");

// 抛出业务异常（带原因）
throw new BusinessException(ErrorCode.D0018, "导出文件写入失败", e);
```

### 前端使用

```javascript
// 拦截器自动处理，无需手动调用

// 如需在组件中精细处理特定错误码
import { ErrorCode, isStockError, handleStockError } from '@/utils/errorCode'

try {
  await addToCart(data)
} catch (error) {
  if (error.errorCode === ErrorCode.C0003) {
    // 库存不足 — 引导用户减少数量
    handleStockError(error.errorCode, error.message)
  }
}
```

### 新增错误码规范

1. 确定所属模块前缀（A/B/C/D/E）
2. 在该模块区间内分配下一个序号
3. 同时更新后端 `ErrorCode` 枚举和前端 `errorCode.js`
4. 更新本文档
